Overview

This table evaluates our Lab 01 website using Norman’s Action Cycle, GOMS model, and KLM operators. Each model highlights different aspects of user interaction and website usability.

Comparison Table

How Norman, GOMS, and KLM apply to our website
Model Focus Application to Website Observations
Norman Cognitive / Descriptive - Header clearly identifies site (goal recognition)
- Navigation menus signal destinations (mapping & affordances)
- Forms provide clear labels (user understands actions)
- Feedback is given via tables and link structure 		Strength: Explains mental model & user understanding
Weakness: Does not predict performance time
GOMS Predictive / Analytic - Tasks like navigating to Contact page or filling a form can be decomposed
- Each task has goal → method → selection rules → operators
- E.g., Goal: submit message → Method: fill fields → Operator: type text → Selection rule: choose which page first 		Strength: Shows efficiency & structure of expert behavior
Weakness: Ignores cognitive feedback & interpretation
KLM Execution-level / Quantitative - Execution of primitive actions can be timed:
• [K] Typing in forms
• [P] Pointing mouse or tapping links
• [M] Mental preparation before choosing a link
• [R] Waiting for page load 		Strength: Predicts task duration for expert, error-free users
Weakness: Does not include high-level planning or errors

Summary

By comparing all three models on our Lab 01 website:

  • Norman: Explains mental processes and usability perception.
  • GOMS: Shows task decomposition and efficiency for expert users.
  • KLM: Predicts execution times for interactions.

Together, these models give a full picture: understanding, structure, and timing of user interactions.
