Norman vs. GOMS – Conceptual Comparison

Comparison of Norman’s cognitive stages with GOMS components
Norman Stage Description (Book Example) GOMS Equivalent Notes / Alignment
Forming Goal User decides what they want to accomplish (e.g., “Send email”). Goal High-level intention corresponds directly to GOMS “Goal”.
Forming Intention Planning the steps needed to reach the goal (e.g., open mail client). Method & Selection Rules Norman’s plan maps to GOMS method; choice between alternatives captured by selection rules.
Specifying Action Deciding what exact action to perform (e.g., click “Compose”). Operator (partially) GOMS abstracts this step; execution-level detail may appear in KLM as [M] or [K].
Executing Action User performs physical actions (mouse click, key press). Operator Direct mapping; KLM further predicts timing of these operators.
Perceiving / Evaluating Outcome User interprets system feedback and evaluates success (e.g., “Email sent?”). Not explicitly modeled in GOMS Norman emphasizes feedback and Gulf of Evaluation; GOMS assumes correct execution without error checking.

Summary

This table shows that Norman’s model focuses on cognitive understanding, planning, and evaluation, while GOMS focuses on predictable, expert-level task sequences. KLM (covered separately) models execution timing for physical operators.
