Comparison of Cognitive Models

Norman, GOMS, and KLM: Explanation vs. Prediction (Dix et al.)
Model Category What It Explains What It Predicts
Norman Cognitive / descriptive User goals, intentions, errors, feedback, gulfs of execution and evaluation ❌ Does not predict execution time
GOMS Predictive / analytic Hierarchical task structure for expert users ✔ Task execution time and method efficiency
KLM Predictive / quantitative Execution-level actions, motor and mental operators ✔ Execution time of primitive actions

KLM Operators

KLM breaks down expert user interaction into primitive operators. Each operator represents a single, timed action. High-level cognitive planning is assumed to have been completed.

KLM Operators (Dix et al., Ch. 12)
Operator Name Meaning Example
K Keystroking Pressing a key (including modifiers) Typing a letter, Delete
B Button press Pressing a mouse button Clicking an icon
P Pointing Moving pointer to a target Move cursor to button
H Homing Moving hand between devices Keyboard → Mouse
D Drawing Drawing with pointing device Drag line to draw shape
M Mental preparation Thinking before executing an action Pause before clicking correct control
R System response Waiting for system feedback Loading screen

Worked Example: Correcting a Character

Task: User sees a single-character error, deletes it, retypes it, and returns to the insertion point.

  1. Move hand to mouse
  2. Position cursor after incorrect character
  3. Return to keyboard
  4. Delete character
  5. Type correct character
  6. Return to insertion point

Corresponding KLM sequence:

H[mouse]
P B[LEFT]
H[keyboard]
M K[DELETE]
K[char]
H[mouse] M P B[LEFT]

Only execution-level actions are modeled in KLM; planning and interpretation are outside its scope.

Key Takeaways

KLM uses seven primitive operators (K, B, P, H, D, M, R) to predict execution time for expert users. It represents the lowest level of GOMS.

Norman’s model focuses on intentions, planning, and evaluation, which KLM and GOMS only partially represent.
